How it works
01

Proxy forwarding

Requests are streamed directly to target servers with zero buffering. Runs on Bun for minimal overhead.

02

Auth injection

Each profile has its own API keys and headers. Clients hit the proxy without credentials — the proxy adds them.

03

Live telemetry

Request counts, latency, and errors tracked per profile. Exportable via OpenTelemetry or the built-in dashboard.

04

Webhook fan-out

Receive incoming payloads and asynchronously dispatch them to multiple destinations instantly with native Meta verification support.
