Notas

  • Scope: admin → apenas login do dashboard; proxy → apenas profiles; both → ambos.
  • 2FA: required obriga TOTP no primeiro login; optional permite saltar; disabled ignora. Admins via LDAP exigem sempre TOTP.
  • A password do bind admin é cifrada em AES-256-GCM com chave derivada da chave RSA do JWT.
  • O filtro deve conter {login}. Default cobre uid, mail e sAMAccountName.

Novo directory LDAP

Ligação
Mapeamento de utilizadores
Deve conter {login}. Default cobre OpenLDAP e AD.
Política
Autorização
Aceita o nome curto do grupo (CN) ou o DN completo. Ex.: DSI-DESENVOLVIMENTO-INOVACAO ou CN=...,OU=...,DC=.... O user só entra como admin se um destes aparecer em memberOf. Vazio = nenhum admin LDAP.
Atribuído automaticamente no primeiro login a users que cheguem por este directory. Vazio = shadow account sem acesso até ser associada manualmente.
Risco: Permite que uma password AD tome posse de uma conta local com email igual. Activa só em directories onde tens a certeza que não há homónimos. Adopções ficam pendentes de confirmação manual em LDAP → Conflitos. Contas locais com 2FA configurado são sempre recusadas mesmo com esta opção ligada.
